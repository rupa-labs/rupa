# Path: The Handcraft Path (Composite) 🧩

The **Handcraft Path** is the ultimate workshop for Artisans who carve specialized systems from raw materials. Unlike other paths that focus on a specific display target, this path is designed for **Bespoke Assemblies**—where you are free to "handcraft" any combination of Rupa atoms into a new, functional masterpiece.

---

## 🏗️ What is Handcrafting in Rupa?

Handcrafting is the act of orchestrating multiple **Atoms** (Tier 1) and **Kernel Systems** (Tier 2) into a specialized tool, library, or engine. 

- **Artisanal Logic**: You can build complex reactive engines that don't necessarily have a UI.
- **Material Oriented**: You can create new UI component collections (custom materials) to be shared across the ecosystem.
- **Adapter-Less Purity**: By default, this path doesn't attach to a specific hardware showroom, allowing your creation to remain pure and portable.

---

## 🎯 Best For:

1.  **Specialized Engines**: Building a custom physics engine, a reactive state manager, or a domain-specific logic bus.
2.  **Shared UI Materials**: Crafting a unique design system or a collection of high-level components for others to use.
3.  **Cross-platform Logic**: Assembling complex business behaviors that need to run identically across Desktop, Web, and Terminal.
4.  **Experimental Raking**: When you want to prototype a new way of connecting Rupa sub-systems.

---

## 🚀 Quick Start

Initialize via CLI:
```bash
rupa create --template handcraft
```

### The Artisan's Workbench
In this path, your `src/lib.rs` or `src/main.rs` is a blank workbench. You decide which **Ports** to implement and which **Kernels** to orchestrate.
