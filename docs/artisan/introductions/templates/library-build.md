# Path: Library Build (Composite) 🧱

The **Library Build** path is the ultimate "Handcraft" zone for Artisans who carve specialized systems from raw materials. This is a blank canvas where you are free to assemble any combination of Rupa atoms and composites into a new, functional masterpiece.

---

## 🏗️ What is a Library Build?

It is a project focused on **Assembly** and **Contribution**. Instead of targeting a specific display hardware, you build the logic and visual "materials" that can be reused across the entire ecosystem.

- **Bespoke Assemblies**: Build custom physics engines, state managers, or domain-specific logic buses.
- **Shared UI Materials**: Create new design systems or high-level component collections.
- **Pure Portability**: By default, this path doesn't attach to a physical adapter, ensuring your creation is agnostic and lightweight.

---

## 🎯 Best For:

1.  **Shared Collections**: Building and sharing new Rupa components on crates.io.
2.  **Logic Engines**: Crafting reusable reactive behaviors for the community.
3.  **Experimental Raking**: When you want to prototype a new way of connecting Rupa sub-systems.

---

## 🚀 Quick Start

Initialize via CLI:
```bash
rupa create --template library
```
